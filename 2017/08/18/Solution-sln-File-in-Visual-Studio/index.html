Solution(.sln) File in Visual Studio

使用Visual Studio创建的项目都包含一个.sln文件，本文简要说明文件的内容及相关作用。

文件说明

在Visual Studio中，解决方案是用于组织管理项目的结构。解决方案将所有项目的状态信息保存在.sln文件和.suo文件中。.sln文件以文本信息的方式，保存着环境需要用于查找和载入项目及其引用VSPackage的持久化数据name-value参数。当一个用户打开解决方案时，开发环境将遍历.sln文件中的preSolution、Project和postSolution信息，载入解决方案、解决方案中的项目、以及解决方案相关联的持久化信息。

开发环境载入流程

参考(MSDN官方文档)[https://msdn.microsoft.com/en-us/library/bb165951.aspx]，为了载入`解决方案`，环境平台将执行以下任务：

  1. 开发环境将阅读Global区域，处理所有标记为preSolution的区域，例如：

  2. 开发环境载入VSPackage，调用其中的接口QueryInterface，并使用区域中的数据调用ReadSolutionProps方法，这样数据就能存储在VSPackage中。开发环境将对每个preSolution区域重复这个流程。

  3. 开发环境迭代存留区域（persistence blocks）的项目。

这份声明包含了唯一的项目GUID，以及项目类型GUID。这些信息是被开发环境用于找到属于解决方案的项目文件，以及每个项目所需要的VSPackage。项目GUID将被传递给IVsProjectFactory以载入相关的VSPackage，然后载入项目。

  1. 基于.sln文件项目区域所包含的信息，环境变量载入每个项目文件。项目自身需要负责构建项目层级并载入内嵌的项目

  2. 在.sln文件所有区域后，解决方案将被显示在Solution Explorer中，并准备好呈现给用户。

如果解决方案中任何实现项目的VSPackage载入失败，将调用OnProjectLoadFailure方法，其它项目将让用户选择是否忽略。如果发生解析错误，开发环境将显示对话框警告用户解决方案损坏了。

总结

.sln文件包含了解决方案所需要的相关信息，包括开发环境的版本要求、属于解决方案的项目信息、载入解决方案所需要执行的相关操作等。在启动时，开发环境将根据.sln文件执行相关准备，完成后将解决方案呈现给用户，否则提供相关的提示信息。
