Grunt入门

什么是Grunt？

Javascript的任务运行器（The Javascript Task Runner）

为什么需要使用任务运行器？
自动化。当执行重复任务如压缩、编译、单元测试等的工作越少，工作就越轻松。只要通过配置Gruntfile文件，任务运行器就能帮助你以几乎零付出的代价完成这些平庸的事情。

为什么使用Grunt
Grunt的生态系统很大，并且每天还在不断增长。基于成千上百的插件可供选择，能够使用Grunt自动完成许多工作。如果没有人构建你所需要的，制作并发布自己的Grunt插件也是一件轻而易举的事情。

Grunt的使用习惯

传统的文件夹结构特征会包含这些文件夹：src、dist和test。

  • src文件夹（有时命名为app）包含创作的源代码库
  • dist文件夹（有时命名为build）包含发布版本，已经压缩的源代码。压缩文件中所有不必要的字符，比如空格、换行、注释，都被移除，但不影响源代码的功能。压缩源代码是很有用的，因为能够减少需要传输的数据量。
  • test文件夹包含测试项目的源代码。

开发功能库并发布新版本时，有一些基础任务是必须执行的。
比如，你希望确保你所撰写的代码达到最好的实际效果，或者说代码不会造成意外的行为。为了确保这一点，你可能需要使用一个工具JSHint。Grunt有一个官方插件grunt-contrib-jshint。尤其，你可能希望确保当你修改代码时，不会打破任何规则和实用效果。因此一个比较好的策略是在每次你执行改动时都检查代码。
为了完成这个任务，我们还会涉及到一个Grunt插件grunt-contrib-watch。当文件增加、修改、或删除时，其能够运行预定义的任务，比如grunt-contrib-jshint。
好好检查你的源代码符合最优结果不足以保证它是稳定以及不包含bug的。为了创建一个鲁棒的项目，你需要对它进行测试。有一些库可以采用，比如QUnit或Jasmine。在Grunt插件中可以使用grunt-contrib-qunit来测试代码。
当需要分发你的工作时，你希望提供一个尽可能小的版本。为了创建一个压缩版本，你需要Grunt插件如grunt-contrib-uglify。进一步，除非你正在开发的项目很小，你将代码划分到多个文件中的可能性是很大的。尽管这对于开发者来说是一个好习惯，但你希望用户只包含一个文件。因此，在压缩代码后，你还希望将源代码们合并成一个单一的文件。为了实现这个目标，你还需要Grunt插件grunt-contrib-concat。

总结一下，通常会用到以下五个Grunt插件：

  • grunt-contrib-uglify
  • grunt-contrib-qunit
  • grunt-contrib-concat
  • grunt-contrib-jshint
  • grunt-contrib-watch

总结

  • Grunt是基于Node.js的自动化任务运行器，能够帮助减轻重复性的代码工作。
  • Grunt生态系统强大，有丰富的插件可供选择
  • Grunt项目中最重要的两个文件是package.json和Gruntfile
  • Gruntfile定义了运行器的配置，由4部分组织：包装函数、任务配置、载入执行、自定义任务
