Git tag的使用

2017-10-09

基本概念

Git Tag故名思义是给Git打标签，可以对某一时间点上的版本打上标签。
在发布软件版本的时候，常用于标记某个特定版本的提交，方便查找和管理。

主要命令

显示标签

git tag 显示所有标签
git tag -l 对标签名称进行特定过滤，如
git tag -l 'v1.4.2.*' 只显示v1.4.2系列的标签

新建标签

Git使用的标签有两种类型：轻量级的（lightweight）和含附注的（annotated）。
轻量级标签是指向特定提交对象的引用（类似于一个Alias，方便记忆）；含附注标签实际上是存储在仓库中的独立对象，有自身的检验和信息，包含标签名字、电子邮件地址、日期、说明等，同时还允许使用GNU Privacy Guard(GPG)来签署或验证。
通常建议使用含附注标签，以便保留相关信息。

轻量级标签实际上就是一个保存着对应提交对象的校验和信息的文件。创建轻量级标签时，不需要相关选项，直接给出标签名答即可：

签署标签

如果有自己的私钥，可以使用GPG来签署标签，只需要把之前的-a改为-s即可
`git tag -s [version] -m ‘[annotation information]’，比如：

验证标签

使用git tag -v [tagname]可以验证已经签署的标签，这个命令会调用GPG来验证签名，需要拥有签署者的公钥，存放在keyring中，才能验证。

后期加注标签

可以对之前某次提交添加加注标签，方法是在创建标签时添加上提交的md5值git tag -a [version] -m '[annotation information] [commit_id]

常用方案

在产品发布里程碑版本后，给其打上一个标签，以便需要时候查看和快速切换。
从本质上讲，就是使用一个tag记录某次提交的语义信息，通过直接用tag而不是commit_id的方法检索到对应的某次提交。
