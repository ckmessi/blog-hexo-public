Azure Application Insights入门

2017-09-11

Application Insights是微软Azure平台的一项服务，声称能够全面监控和了解服务的行为、可靠性和性能。
可以这样理解，当一项应用添加了Application Insights插件后，相当于给自己的应用添加了许多监控机制，可以通过Azure平台页面直接监测服务的运行状况。类似于阿里云平台查看云主机吞吐量，或自行使用tail -f x.log命令监控日志信息。

基本概念

首先需要理解以下几个概念：

  • Azure Application Insights：微软推出的服务监控插件，可自行配置安装到应用中。
  • Microsoft Azure：微软推出的云平台，用于监控配置了Application Insights的服务资源。
  • InstrumentationKey：检测密钥，一项Azure上的Application Insights资源对应唯一的密钥，当项目配置密钥后，将直接将该项目应用绑定到密钥所标识的Azure资源上。

Application Insights在ASP.NET中的使用

设置InstrumentationKey

Application Insights的基本使用方法可以参考微软官方文档。
主要包括安装相关SDK，初始化Application Insights配置等流程。
最重要的步骤是添加InstrumentationKey，事实证明所有流程的最终结果本质上是在ApplicationInsights.config文件中添加该配置项，这样应用就能够连上Azure监控平台。

由于不同部署平台可能需要使用不同的InstrumentationKey，比较好的方案是在程序启动后执行的代码中读取配置文件动态添加。
参考文档，可以在项目的启动添加中动态添加InstrumentationKey。

设置cloud_RoleName

cloud_RoleName的作用是区分使用同一个InstrumentationKey的不同组件（Components）。例如，在同一个应用下，可能有多个组件组成，此时如果希望对一些性能指标、日志记录进行筛选，则需要给每个组件添加唯一的cloud_RoleName，在Azure监控平台上的”搜索”中可以添加cloud_RoleName的筛选条件。

参考官方文档1和cloud_RoleName的覆写方法，可以在程序启动时为应用配置特殊的cloud_RoleName。
