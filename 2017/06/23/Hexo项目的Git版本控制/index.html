Hexo项目的Git版本控制

Hexo项目目录解析

使用Hexo能够快速地创建个人博客。
但如何进行Hexo项目的版本管理控制呢？换句话讲，Hexo项目中哪些内容应该添加到版本库中？

使用hexo init [project_name]命令能够快速创建一个名称为[project_name]的文件夹，里面包含了以下文件：

.
├── .gitignore
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

其中：

  • .gitignore是git仓库忽略配置文件
  • _config.yml用于修改相关配置
  • package.json用于管理依赖库
  • scaffolds用于模板管理
  • source文件夹为添加新博客的内容
  • themes存储主题

Hexo init命令的实质

经过观察和分析，其实执行hexo init [project_name]命令的实质内容，可以等价为以下几个步骤：

即从github的hexo-starter仓库中clone一份初始模板；
然后执行npm install，按照package.json中的内容安装对应依赖库；
最后把git仓库的相关信息删除。

Hexo项目需要关注的文件

因此，对于我们自己的Hexo项目，在进行版本控制时主要采用以下方案：
将以下内容添加到版本管理中，

  • .gitignore 基本不会修改，但需要加入版本控制
  • _config.yml 修改基础配置使用
  • package.json 修改相关依赖库使用
  • scaffolds 修改模板文件使用
  • source 添加/编辑博客文件使用
