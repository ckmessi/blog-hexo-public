Git克隆项目时SSH/HTTP方式的区别

在使用Git获取仓库中的项目时，通常可以选择用SSH或HTTP方式。
两种克隆方式对应的URL也不一样。以Github为例，选择Clone with SSH时，对应的URL为：

git@github.com:ckmessi/ckmessi.github.io.git
而选择Clone with HTTPS时，对应的URL为：

https://github.com/ckmessi/ckmessi.github.io.git
两种方式的区别在于：

使用SSH方式时，需要预先配置ssh key相关信息，克隆过程实质上建立了一个SSH连接（就像远程免密登录服务器一样）；如果在配置ssh key时没有设置密码，则每次push时不需要验证密码。
使用HTTP方式时，可以不经任何验证克隆该项目的数据，但在每次push的时候，都需要验证用户名和密码。
类似于SSH是一项永久性操作，而HTTP只是快速构建临时可读的方案。

因此，在实际工作中，如果要频繁地获取并提交修改，应该使用SSH方式；如果只是将项目克隆下来随意浏览，不需要提交修改操作，则使用HTTP更方便。
