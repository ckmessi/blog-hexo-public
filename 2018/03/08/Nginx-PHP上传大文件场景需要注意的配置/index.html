Nginx+PHP上传大文件场景需要注意的配置

在使用nginx+php作为后台服务器时，如果遇到需要上传文件的业务逻辑，需要注意两者对文件的限制大小。

Ngnix配置

首先，有可能遇到的情况是，前端网页上传文件时报错了；而后台服务器没有接到访问请求。这是由于nginx对客户端的body大小的限制，请求会中断在nginx，无法到达php。默认情况下，上传大文件可能造成错误413 Request Entity Too Large的情况。

在nginx.conf中，添加一行：

client_max_body_size的默认值为1m，上述语句表示客户端可以上传的最大body大小为20Mb。

至于在哪个层级下设置，查找到一篇博客是这样说的：

可以选择在http{ }中设置：client_max_body_size 20m;
也可以选择在server{ }中设置：client_max_body_size 20m;
还可以选择在location{ }中设置：client_max_body_size 20m;
三者到区别是：http{} 中控制着所有nginx收到的请求。而报文大小限制设置在server｛｝中，则控制该server收到的请求报文大小，同理，如果配置在location中，则报文大小限制，只对匹配了location 路由规则的请求生效。

PHP配置

解决nginx的配置问题后，还有可能在php代码中出现获取不到提交文件的问题。这是由于php对于文件大小也有限制。
