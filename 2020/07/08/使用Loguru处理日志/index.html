花括号风格的字符串格式化

Loguru偏向于使用更优雅的 {} 格式化而不是 %，日志方法的效果与 str.format 是等价的：

线程或主函数里的异常捕捉

使用 catch 装饰器，能够确保一个方法内发生的异常被正确地传递到 logger 中。

更漂亮的颜色

Loguru自动添加了一些颜色，同样你也可以自定义配置喜欢的颜色，使用 markup tags 相关的配置。

异步、线程安全、多进程安全

所有添加到 logger 中的处理器都默认是线程安全的，它们不是多进程安全的，但可以使用 enqueue 参数来确保日志内容的完整性。

描述完整的异常信息

记录异常信息对于追踪bug而言是至关重要的，但如果你不知道为何失败了就没啥用。Loguru 能够展示完整的调用堆栈来帮助定位问题。

只要指定了 backtrace 和 diagnose 在捕捉异常时，只需要打印关键信息，其余堆栈信息会自动添加。
这两个参数默认值都是 True，如果设置为 False，可以参考查看在日志中减少的部分。

按需制定的结构化日志

使用 serialize 参数，每条日志信息将会被转换成 JSON 格式的字符串。

计算耗能较高的方法的懒加载

通过 opt 方法指定 lazy=True 可以避免生产环境的性能问题。

自定义级别

针对脚本和库的适配

在一个脚本中使用日志是很容易 的，可以直接使用 configure 方法配置。在一个库中使用 Loguru，记得不要使用 add 而是使用 disable，如果开发者希望看到相关的信息，它可以使用 enable 来重新启用

方便地解析

Loguru 提供 parse 方法，帮助在日志中使用正则表达式提取相关信息

详情的通知器

Loguru 可以轻松地结合一些通知器的库，用于发送电子邮件，当程序遇到预期之外的异常时，通过各种方式通知开发者

更多细节可以参考 Loguru官方项目说明。

总结

Loguru 可以更加方便、灵活地配置日志处理相关的逻辑，实现完善、丰富、个性化的日志管理。
