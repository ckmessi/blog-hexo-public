GitLab中添加代码覆盖率报告数值

背景

在使用 GitLab 持续集成的特性时，可以在 CI/CD 的过程中添加输出测试代码覆盖率，并实时集成至项目主页面上。

方法

整体方法挺简单的，主要分为三个部分：
1、在 CI/CD 流程中添加输出覆盖率的操作；
2、在项目配置选项中添加对覆盖率的监听；
3、将覆盖率徽章添加到 READMD.md 中；

测试阶段输出

通常在 GitLab 的 CI/CD 流程都会经过 build，test 和 deploy 三个阶段。
在 test 阶段，通常是配置相关的单元测试或集成测试语句，判断测试是否通过。
以 Python 项目为例，通常会使用 pytest 执行测试。

这里以 Python 中常用的 coverage 包为例，完成对测试代码覆盖率的输出。

配置

在项目的 [Setting] -> [CI / CD] -> [General piplines settings] 中可以找到一项配置 Test coverage parsing，它要求填写一个正则表达式，用于寻找在测试过程中输出的表示覆盖率含义的值。

页面中还给出一些示例，对于 coverage 工具而言满足的是 ^TOTAL.*\s+(\d+\%)$ 格式。
输入后点击 Save change 按钮即可。

总结

在使用 GitLab CI/CD 的流程中，可以添加测试代码覆盖率的操作，并将覆盖率结果实时地显示在项目主页中。
其中需要使用到的工具主要是依据不同语言选用的测试代码覆盖率统计工具，以及在 GitLab 配置页面中的相关配置操作即可。
