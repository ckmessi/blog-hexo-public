初识TorchServe

背景

TorchServe 是一个灵活易用的工具，用于将 PyTorch 模型服务化。

上述语句中的 服务化 包含了这些特性：

  • 模型仓库的构建
  • 模型部署的注册、维护和管理
  • 推理请求的处理

其中一些专用术语的解释如下：

  • 前端：处理请求/响应的组件，能够处理来自客户端的请求和响应，并管理模型的生命周期。
  • 模型工作器：负责运行实际的在模型上进行推理的操作；
  • 模型：模型可以是一个 script_module（JIT存储模型）或 eager_mode_models，提供自定义的前/后处理数据流程。模型能够从云存储或本地服务器上加载；
  • 插件：自定义的入口或授权网关，或者能够在 TorchServe 启动时载入的批量算法；
  • 模型仓库：一个放置着所有可加载模型的目录；

对应的，在使用 TorchServe 的过程中通常包含几个流程：

  • 归档模型
  • 启动主进程，将模型服务化
  • 获取模型的推理结果

使用TensorRT-7.2.1时遇到的坑

背景

初入 TensorRT 的坑，不太熟练，在使用过程中，遇到抛模型的构建版本与当前运行环境版本不一致。

经过排查，发现是系统所装的 TensorRT 是 7.2.1 的，而 Python 包安装的 nvidia-tensorrt 是 7.2.3 的，通过查看系统软件和Python包的命令可以检查：

解决方案

直接的解决方案是将 Python 包中的 nvidia-tensorrt 换成 7.2.1，但是安装过程中会抛找不到对应版本的错误：

在官网查找后发现，原来 7.2.1 只支持 Python 3.6 版本，而当前使用的是 Python 3.8，因此无法找到对应安装包。

需要重新使用一个 Python 环境，设定预置 Python 版本为 3.6。

尝试直接安装：

然而不带版本号却能装，装的是 7.2.3。
机智地通过装 7.2.3 的链接找到了对应 .whl 文件的 仓库地址。到对应的目录下找找看，原来是它版本号需要4位，如果写全了 7.2.1.6 是可以安装的。

后续对应安装 pycuda、opencv-python 和 torch 相关的包即可。

总结

使用 TensortRT 时可能涉及到两个环境，构建模型使用的可能是系统的版本，而运行时有可能使用的是 Python 包，需要确保两者统一。

  • https://developer.download.nvidia.cn/compute/redist/nvidia-tensorrt/
  • https://docs.nvidia.com/deeplearning/tensorrt/archives/tensorrt-721/install-guide/index.html

PHP中strtotime月份加减的坑

在使用 PHP 撰写与时间日期相关的操作时，可能会用到 strtotime 这个方法。

比如：

此时的 $yesterday_date 变量的值为 2021-03-28。

同理，也可以操作月份，比如：

这时 $date_1 和 $date_2 的值分别是 2020-12-15 和 2021-02-15， 看起来挺正常的。

但是，如果遇到月末的情况，就非常尴尬：

这时 $date_1 和 $date_2 的值分别是 2021-03-01 和 2021-05-01， 看起来就有一点反直觉了。

这是由于 strtotime 执行的月份操作是直接添加，比如对于 2021-03-29 的 -1 month 得到的是 2021-02-29，然而并没有这个日期，就被直接进行转换变成了 2021-03-01；同理，对于 2021-03-31 的 +1 month 得到的是 2021-04-31，同样并没有这个日期，就被直接进行转换变成了 2021-05-01。
