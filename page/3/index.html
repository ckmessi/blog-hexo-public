使用crontab添加特定用户执行的定时任务

背景

使用 Linux 系统中的 crontab 功能，可以为系统添加定时任务。

比如使用 crontab -l 可以查看当前用户设定的定时任务，使用 crontab -e 可以编辑定时任务。具体定时任务的格式在这里不详加叙述，这篇文章主要针对执行定时任务的用户。

事实上，Linux 是一个多用户系统，每个用户执行 crontab -l 看到的定时任务内容是不同的，都是属于其自己的定时任务。而这些定时任务的操作者，也会被自动选中为添加该定时任务的用户。
但有的情况下，定时任务如果由普通用户创建，很可能会造成问题。比如一个定时任务如果写日志文件，而这个日志文件正常情况下是由项目的服务启动者（比如 www-data）来执行写入操作，那么就有可能会出问题。
现实项目中就出现这样一种情况，Laravel 项目的错误日志在日期分片后会写在 storage/logs/ 目录下的 laravel-yyyy-mm-dd.log 文件中。如果有一个普通用户 x 创建的定时任务在每天凌晨时执行，且执行过程中写日志了，就会自动创建一个所有者是 x 的 laravel-yyyy-mm-dd.log。而当框架代码正常情况下需要往这个日志写内容时，由于文件是 644 权限，托管 Laravel 项目的 nginx 用户 www-data 无法写文件，就会抛出异常。

解决方案

将定时任务从个人列表中删除，由全局添加，并且在添加时指定执行用户。

在所有用户个人的 crontab 列表之外，还有一个全局的定时任务列表，内容在 /etc/crontab 文件中。
编辑这个文件需要 root 权限，且这个文件的内容格式与普通用户的略有不同。
普通用户的格式为以下格式，前5列是执行频率，然后就是执行语句：

全局的格式在前5列之后，多了一列执行用户，然后才是执行语句：

因此，只需要将原有的定时任务内容，搬到全局 /etc/crontab 文件中，然后在最后执行语句之前加上一列执行用户信息即可。
