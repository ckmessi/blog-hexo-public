简介

Alembic 是一个基于 SQLAlchemy（一种SQL工具包及ORM映射管理器）的轻量级数据库迁移工具，使用 SQLAlchemy 作为引擎基础，为关系型数据库的创建、维护以及修改维护的变化提供了相关脚本。

安装

Alembic的安装通常是一个项目安装的本地设置，通常假定使用虚拟环境。尽管 alembic 命令行工具不管在哪安装都能够正常使用，但项目本地安装的合理性在于 Alemic 命令行工具的大部分关键操作都是通过一个 Python 文件 env.py 执行，这个文件是项目安装过程中的一部分通过 alembic init 命令运行产生的。 env.py 文件的目的是为迁移过程建立数据库连接和可选的模型定义，后者尤其依赖于导入项目本身的相关模块。

在安装流程中提到，最好将 Alembic 安装在当前目录项目相同的 Python 执行目录中，比如使用 Python virtualenv 构建。这样当 alembic 命令执行的时候，alembic 触发的 Python 脚本（即对应的 env.py 脚本）能够直接访问应用的模型。这不是严格限制条件，但在大多数场景中会这样操作。

  • script.py.mako 是一个 Mako （一种 Python 模板库）模板文件，用于生成新的迁移脚本，即 versions/ 目录中的新文件。通常情况下不需要修改，但也可以自定义 upgrade() 和 downgrade() 方法模板，这样在多数据库环境中就可以扩展出 upgrade_engine1() 和 upgrade_engine2() 等方法；
  • versions 目录保存了每一个单独的版本脚本，它使用 GUID 标识每个文件。在 Alembic 中，版本脚本的顺序与脚本自身的指令相关，理论上是可能把一部分版本文件与另外一部分隔离开，允许从不同分支合并的迁移能够被手动合并、执行。

简介

一直在寻找一种撰写项目文档的最佳方式，希望使撰写文档更加便捷、查看文档更加方便，同时更好地管理文档的版本控制。
传统方案可能是使用 Word 等办公软件撰写文件，然后把文件发送给对方。这种方法有诸多不便：

  • 对方必须下载或缓存文件才能查看，造成设备空间浪费；
  • 跨平台友好性不足，对方的操作系统如果与创建者不同，则无法打开或无法编辑；
  • 文档不断修改，只能以文件名后缀 _v2、_v3 用于区分版本，相对杂乱；

现有通用的方法可以使用类似石墨文档等在线文档协同编辑工具。

Mkdocs通过配置文件管理主题样式，有许多成熟的样式模板可供选择，也完全可以自己构建一套样式。

Laravel批量insert时数组维度的一个坑

背景

在 Laravel 官方文档中，对于数据库插入操作 insert 的描述比较简洁：

You may even insert several records into the table with a single call to insert by passing an array of arrays. Each array represents a row to be inserted into the table:

这是一项非常实用的功能，意味着批量操作的时，可以通过执行一条 SQL 完成大量数据的快速插入。

在 Laravel Database Query Builder insert的效率问题 文章中，提到了曾经遇到的效率问题。这是由于 Laravel 底层代码在组织 SQL 语句时对数组进行了 array_merge 操作，导致对于长度为 n 的数组时间复杂度变成了 O(n^2) 。

今天又遇到了一个坑，即数组中每个元素的维度必须是相同的，否则有可能导致最终插入数据库的数据列与列之间发生错乱。

原理分析

还是从 Laravel 框架中 Query\Builder.php 源文件的 insert 方法入手，逐步查找造成问题的原因。

这里要走两条线，一条件是生成 SQL 语句中的占位符，一条线是生成绑定的数据。

支线1-1）在 Query\Builder.php 中的 insert 方法，是代码的入口，对参数 $value 按照键进行排序后，封装成相关的 SQL 语句调用数据库连接的插入方法 $this->connection->insert：

支线1-2）在组织 SQL 语句的过程中，关注 Query\Grammars\Grammar.php 文件的 compileInsert 这个方法。
它主要执行操作主要有三步：

  • 先取出所有数据第一个元素的键列表作为 column 列信息（reset 方法将数组内部指针指向第一个元素，并返回第一个元素）；
  • 将整个数组按照值进行映射，组织成一个很长的 SQL 语句；
  • 根据上述两个信息，拼接成一个实际的 insert into ... SQL 语句。
