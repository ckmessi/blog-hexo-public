VSCode使用SFTP完成代码远程同步

背景

工作中许多场景下需要在服务器上运行代码，这往往是因为服务器具有本地开发环境所不具备的某种资源（比如GPU显卡等）。在这种情况下，小的代码修改可以通过 vim 等文本编辑工具完成，但涉及到较多代码改动时，可能使用 IDE 会更有效率。

使用 VS Code 的 SFTP 扩展可以方便地实现本地代码与服务器代码的同步。

安装

在 VS Code 界面中使用 Ctrl + Shift + X 打开 Extension 面板，搜索 SFTP，安装。

SFTP
SFTP/FTP sync
liximomo

使用

在 VS Code 界面中使用快捷键 Ctrl + Shift + P，输入 SFTP: Config ，会在当前项目的根目录下创建一个 .vscode 目录，并在其中创建一个 sftp.json 文件，这个文件将包含所有的配置信息。

换句话说，针对不同的项目，需要有不同的 sftp.json，而这个配置文件中只需要选择当前项目对应的远程服务器信息即可。

样例配置

基本是平时 SSH 配置所需要的相关信息，通常会使用 password 设置密码，或使用 privateKeyPath 设置私钥存放位置，否则在上传时需要输入密码进行身份验证。

完成后，打开任何文件，按 Ctrl + Shift + P，输入命令 SFTP: Upload ... ，可以选择上传当前打开的文件、目录或整个项目。而配置中的 uploadOnSave 则指定了在编辑文件保存后，自动触发上传操作。

设置多个 Profile

有时本地开发的代码需要同步到不同的服务器上，这时候可以使用多个 profile 的功能，添加对应的 profile 属性区块，并在上传时使用 SFTP: Set Project 选择特定的project` 即可：

设置跳转

如果在当前机器无法直接访问某台远程服务器，可以使用 SFTP 的 hop 功能，具体而言就是在配置中添加一个跳板机，完成转发。

总结

拥有可视化的 IDE 对于编程效率会有较大的提升，而开发环境有时候又不具备这样的条件。使用 VS Code 的 SFTP 扩展可以快速实现本地代码与服务器代码的同步。
