初识FastAPI

简介

FastAPI 是一款基于 Python 3.6+ 的高效 Web 应用框架，能够快速搭建 API 服务器。
在 Python 语言范畴内，FastAPI 可以作为一个快速搭建后台服务器的框架选择。

打开浏览器，访问 http://127.0.0.1:8000，可以看到简洁的 JSON 格式返回值：{"Hello": "World"}。
如果在浏览器中访问 http://127.0.0.1:8000/docs，就可以看到一个漂亮的接口文档。在这份文档中列举了所有可以访问的接口，以及接口的访问方式、参数、返回值、错误信息等。甚至可以直接在页面上点击 Try it out 按钮执行这个 API 请求。

特性

关于 FastAPI，它最重要的特点就是简洁易用，另外它还有一些特性是令我很惊喜的。
比如集成了 OpenAPI 和 Swagger UI，能够对每个路由自动生成相关的格式文档，这样基本可以不需要额外撰写接口文档，只需要按照相关格式正确定义数据类型即可。
在 Swagger UI 提供的文档页面上甚至可以直接执行 API 请求，减少了以往调试过程中需要使用页面或 POSTMAN 等请求发送工具的麻烦。
FastAPI 会使用 Pydantic 进行数据的格式验证，这部分可以减少许多检查请求参数合法性的代码。

基本框架搭建

通常搭建一个后台服务器，最基础的部分就是数据库表的迁移和特定对象的CRUD。
在 FastAPI 中，可以使用 SQLAlchemy 提供的 ORM 完成数据模型的定义及数据库操作，使用 Alembic 完成数据库表的迁移及版本控制，此前撰写的 @app.get("/") 方法即是路由，它所修饰的方法可以理解为传统意义上 Controller 完成的功能。在上述基本功能实现后，一套数据从创建、写入到读取的流程就走通了，能够快速搭建一个可用的 API 服务器。
而 FastAPI 本身还有很多功能，值得挖掘。
