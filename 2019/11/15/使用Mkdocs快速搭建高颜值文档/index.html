使用Mkdocs快速搭建高颜值文档

简介

一直在寻找一种撰写项目文档的最佳方式，希望使撰写文档更加便捷、查看文档更加方便，同时更好地管理文档的版本控制。
传统方案可能是使用 Word 等办公软件撰写文件，然后把文件发送给对方。这种方法有诸多不便：

  • 对方必须下载或缓存文件才能查看，造成设备空间浪费；
  • 跨平台友好性不足，对方的操作系统如果与创建者不同，则无法打开或无法编辑；
  • 文档不断修改，只能以文件名后缀 _v2、_v3 用于区分版本，相对杂乱；

现有通用的方法可以使用类似石墨文档等在线文档协同编辑工具。
这里介绍一种更编程化的方式，使用 Mkdocs 快速生成文档。

解决方案

Mkdocs 是一款快速、简洁、直接的高颜值静态页面生成器，能够快速构建项目文档。

文档源文件以 Markdown 格式撰写，通过一个简单的 YAML 文件进行配置。在撰写文档的过程中，可以实时地预览生成的文档效果。

Mkdocs通过配置文件管理主题样式，有许多成熟的样式模板可供选择，也完全可以自己构建一套样式。

由于 Mkdocs 能够生成静态页面，因此文档可以被方便地部署在 GitHub 页面、Amazon S3 或其它任何地方。

在 mkdocs.yaml 文件中，默认只有一项配置项 site_name，修改之后会影响页面的标题。

添加文档

直接在 docs 目录下添加一个 Markdown 格式的文件，即可添加一份文档。
在默认主题中，不同文档标题会在最上方的导航栏显示，当前文档内部结构会在左侧目录树显示。选择不同的样式模板后，可以有更自由的显示方式。

总结

使用 Mkdocs 能够快速简洁搭建一套项目文档，撰写写只需要在 Markdown 文件中整理好文档内容，并选择合适的主题风格，就可以快速预览或部署文档。
