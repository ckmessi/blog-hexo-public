Composer管理依赖包限制PHP版本

背景

在工作中遇到一个问题，相同的项目在开发机器上可以运行，在测试服务上却抛出错误。
很可能是由于开发机器和测试服务器的 PHP 版本不一致导致的。
比如，在开发机器上可能PHP版本是 7.1.25，而测试机器上是 7.0.30。

在这种情况下，在开发机器上可以运行的项目，只有当部署到测试机器时，才发现无法运行。
这种流程不够完善，应该在开发的过程中，就对项目的 PHP 版本有一定约束。
