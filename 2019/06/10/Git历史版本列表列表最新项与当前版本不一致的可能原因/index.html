Git历史版本列表列表最新项与当前版本不一致的可能原因

背景

在使用 Git 的过程中，通常来讲分支中一个文件的历史版本中最新的那个版本，将是当前分支的文件当前内容。
然而在实际工作中，出现了意外现象：从 GitLab 页面上看，当前 dev 分支中文件A的历史版本按照时间倒序有 v5-v4-v3-v2-v1，然而文件A显示的当前版本却是 v3，实际内容也是版本 v3 提交后的内容。 （GitHub页面上的历史版本只会显示到 v3）
这种现象非常令人困惑，经过和好朋友的一番推理和测试，推测出可能存在的一种造成该现象的原因。
