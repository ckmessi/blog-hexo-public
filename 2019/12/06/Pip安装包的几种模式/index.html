Pip安装包的几种模式

pip 是 Python 的包管理工具，能够便捷地提供 Python 包的查找、下载、安装、卸载功能。
这篇文章主要整理使用 pip 安装的几种方式。

使用 pip install 可以快速地安装一个 Python 包，比如最简单的方式：

这种情况下，默认将从 PyPI （一个 Python 包中心仓库）中获取相关资源并安装。

The Python Package Index (PyPI) is a repository of software for the Python programming language.

除此之外，还有若干种其它安装方式，根据不同场景可能会适用：

  • 从私有仓库安装
  • 从 Git 仓库安装
  • 从安装文件安装

从私有仓库安装

这种方式适用于企业内部开发场景，通常可以搭建一个企业局域网内的 PyPI 私有仓库，然后指定安装源。

从安装文件安装

一个 Python 包项目可以直接将相关源码编译打包到一个安装文件中，然后在目标环境中可以直接使用这个安装文件进行安装。
这种情况大多发生在目标环境没有网络的场景，或相关源网络条件较差时可以手动下载相关文件执行安装。
安装文件也分几种格式，比如相对旧的 Egg 格式以及常用的 Wheel 格式，它们本质上都是一个压缩包，Wheel 的出现是为了替代 Egg ，被认为是目前 Python 二进制包的标准格式。

两种打包方式的区别在于 setup.py bdist 创建一个构建的发行版，主要包含相关二进制文件，但没有 setup.py，通常用于上线使用；而 setup.py sdist 创建了一个源代码分发，它包含 setup.py 和相关源文件，可以在任何平台上重新编译所有内容。

总结

使用 pip install 能够快速安装 Python 包，根据不同的使用场景也可以使用不同的安装方式进行安装。
