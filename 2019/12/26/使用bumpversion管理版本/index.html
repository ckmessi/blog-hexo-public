使用bumpversion管理版本

bumpversion 是一个小型的命令行工具，用于简化应用发布时的版本控制。它主要能够完成以下几个功能：

  • 根据命令升级版本号；
  • 自动替换项目相关文件中的版本信息；
  • 自动创建一次 Git 提交；
  • 为这一次提交打上版本升级的标签；

使用上述 bumpversion 命令时，如果原始版本号为 1.2.3，则使用 bumpversion patch 会升级到版本 1.2.4，使用 bumpversion minor 会升级到版本 1.3.0，使用 bumpversion major 会升级到版本 2.0.0。

option 是配置参数的输入位置，可选内容有许多，具体可参考 官方文档，而 file 表示了将要修改的文件（替换文件中的版本信息），如果没有提供的话，则会查找配置文件中 [bumpversion:file:...] 列表。

配置项

所有配置信息既可以在命令行中指定，也可以通过配置文件的方式指定。通常使用 bumpversion 的项目会在根目录包含一个 .bumpversion.cfg 文件，这样在根目录执行 bumpversion 命令时，会自动根据这个文件加载相关配置。

配置项中常用的配置有以下几类：

  • 全局配置：使用 bumpversion 命令的通用配置；
  • 版本部分指定配置：关于版本形式的相关配置；
  • 文件指定配置：修改文件内容的相关配置

全局配置

全局配置主要就是以下几类，都比较实用，可以关注

  • current_version：当前版本必须指定，没有默认值；
  • new_version：新版本号，可以直接指定，如果没有指定则会根据命令自动生成；
  • tag：是否创建标签，默认为 False，用于标识项目关键版本，并在项目代码的持续集成中扮演关键作用；
  • tag_name：标签名称，默认为 v{new_version}，可根据需要自定义修改；
  • commit：是否创建提交，默认为 False，适用于 git 或 Mercurial 工具；
  • message： 在创建 commit 时的消息

版本部分指定配置

一个版本字符串通常包含一个或多个部分，比如版本 1.0.2 包含了 3 个部分，通过 . 符号分隔。默认情况下这些部分被命名为 major、minor 和 patch ，在 bumpversion 中可以使用 parse/serialize 参数自定义配置。
这部分内容不常用，通常使用遵循默认的版本升级逻辑即可。

总结

bumpversion 是一款基于 Python 开发的命令行工具，使用 bumpversion 可以快速完成项目中版本的升级和管理，减少繁琐的手工操作。
