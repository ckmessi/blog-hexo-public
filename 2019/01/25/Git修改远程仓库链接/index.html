Git修改远程仓库链接

2019-01-25

在使用 git 的过程中，有时候需要修改远程仓库链接。
可能是由于该 git 项目一开始是使用HTTPS的方式clone的，但后来发现经常需要push和pull，每次都要输入用户名密码很烦。
可能是由于临时在外网中，通过SSH转发远程能访问到内网的 git 仓库，但对应的url发生了变化。
可能是存在多个上游（fork的项目），需要额外添加一个上游链接的。

修改方式

通常修改远程仓库链接可以有两种方式：

  • 删除原链接，添加新链接
  • 直接修改原链接

使用命令 git remote -v 可以查看远程仓库地址。
使用命令 git remote rm origin 可以删除远程仓库地址，此时再使用git remote -v查看应该显示为空。
