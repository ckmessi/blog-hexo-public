HTML部分

CSS部分

在 CSS 部分，只是简单地设置了元素的宽高信息、背景色等信息。

JS部分

对于 JS 代码，主要可划分为三个部分。
第一部分，使用 interact('.draggable').draggable({...}) 语句创建了一个 draggable 的操作对象，其中绑定了类名称 draggable ；类似的，如果希望创建一个类名称为 class1 的拖拽区域，则可以使用 interact('.class1').dragzone({...}) 语句。
可以看出，对于 interact.js 库的主要使用语句可以抽象成以下语句：

其中 action 可以是许多，除了 draggable 指定可拖拽对象内容，dragzone 指定拖拽区域属性，还可以使用 resizable 指定拖拽元素放大缩小属性，以及 on 指定各种监听函数。

第二部分，声明了一个函数 dragMoveListener ，函数的操作主要是根据目标元素 event.target 当前的坐标属性 data-x 和 data-y ，加上当前拖动的偏移量 event.dx 和 event.dy ，更新目标元素的新坐标属性，并通过 translate 语句设置目标的实际显示位置 target.style.webkitTransform 和 target.style.transform 。其中 webkitTransform 对应的 css 属性实际是 -webkit-transform ，表示对 safari 浏览器的支持。
这个函数 dragMoveListener 将作为 draggable 对象属性中 onmove 的回调函数，这样每个拖拽元素都将根据 dragMoveListener 中的操作，也是最终拖拽效果体现的原因。

第三部分，设置 window.dragMoveListener = dragMoveListener; 。 这个语句设置全局的 dragMoveListener ，通过注释可以看出，是为了后续 resizing 和 gesture 示例代码中会使用到的。

总结

可以看出，使用 interact.js 的方法主要是通过 interact().action 语句指定目标元素，并通过 interact.js 提供的许多API设置拖动中的各种属性。
通过修改 dragMoveListener 方法，可以控制多个元素同时拖拽（例如拖动父节点时，子节点跟随移动）。
