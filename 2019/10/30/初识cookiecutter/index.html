初识cookiecutter

背景

通常我们在创建项目时，首先需要进行一些重复性的劳动，比如 Web 项目按照 MVC 设计风格拆分出数据层和视图层，并有一个文件夹包含存储路由信息的代码文件。
这类常见的代码组织方式及风格，通常被称为项目框架的基本结构。项目框架的基本结构还通常结合了基本的配置文件、测试代码、构建配置、持续集成等信息，极大降低初学者的入门门槛，也避免了不同开发人员之间习惯风格不同导致的差异。
在一些主流的项目框架中，通常IDE会提供一套标准创建项目的流程向导（Wizard），比如在创建一个 Spring Cloud 项目的时候，使用 JetBrain 的产品 IDEA 可以跟随一套向导快速完成项目框架的创建。
然而在 Python 项目中，主流的编程环境 PyCharm 貌似没有类似功能，因此大多数初学者常用的方式是把一个成熟的项目拷贝一份，然后在此基础上进行修改。
在这种场景下，这种操作的实质效果，就是以这个 成熟的项目 为模板，创建一个新的项目。
这就是 cookiecutter 能够标准化进行的操作。

简介

cookiecutter 是一个命令行工具，用于根据项目模板创建基础的项目框架。

cookiecutter 本身是基于 Python 语言撰写的，因此通常被用于生成各类 Python 框架（如 flask 等）。但 cookiecutter 的使用范围并不局限于 Python 语言，只要提供相关的项目模板，它可以用于生成任何语言的项目，包括但不限于 Python、JavaScript、CoffeeScript 等。
在 GitHub仓库主页 ，描述了它的许多特性：

  • 跨平台：支持 Windows，Mac 和 Linux；
  • 能通过Python2.7/3.5/3.6/3.7 和 PyPy 使用（使用 cookiecutter 并不需要了解 Python 代码）
  • 项目模板可以支持任何编程语言或标记格式
  • 简单的命令行
  • ……

总之，使用 cookiecutter 能够便捷地基于 项目模板 创建一个项目。

总结

cookiecutter 是一款基于项目模板创建新项目的命令行工具。
普通开发者知道如何使用它创建新项目即可。对于项目架构师或高级造轮子的工程师，需要知道如何按照其语法创建一个项目模板，给其它开发者作为模板创建新的项目，这样能够控制团队中项目风格的统一性。
