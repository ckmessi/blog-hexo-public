Java的final关键字

在Java中，关键词 final 通常表示一个 不可改变 的含义。
final 关键字只能用于修饰变量、方法或类，依据其出现的上下文不同，可能有细微的差别。

从使用场景出发，通常来说使用关键词 final 可以有以下几种方式：

  1. 变量
    • 局部变量
    • 成员变量
    • 参数
  2. 方法
  3. 类

一、修饰变量

当变量使用 final 关键字修饰时，意味着它是“不能被修改的”。
如果被 final 关键字修饰的变量是基础类型，则这个变量的值不能改变；
如果被 final 关键字修饰的变量是引用类型，则这个引用的地址的值不能修改，但引用指向的对象里内容还是可以修改的，比如可以从 final 的数组和 final 集合中添加或移除元素。
这里所说的变量通常会存在于三个场景中：

1. 局部变量

对于用 final 修饰的局部变量而言，只能初始化一次。

其中最后一行代码会显示 Cannot assign a value to final variable 'c'。

2. 参数变量

对于用 final 修饰的参数变量，意味着在函数执行过程中，这个参数不可以被修改。

3. 类的成员变量或静态成员变量

对于用 final 修饰的类的成员变量而言，必须进行初始化，且只能初始化一次，通常有以下三种方法：

  1. 在声明时直接初始化，这种方式是最常用的。（如果没有被初始化，则称为 空白的 final 变量），后面两种方式都可以用于初始化 空白的 final 变量）；
  2. 使用 实例初始化代码块 或 静态代码块 进行初始化操作，分别针对成员变量和静态成员变量；
  3. 使用构造函数初始化，如果有多个构造函数，所有构造函数都要进行初始化操作。

一种好的编程风格是用大写字符命名 final 变量，使用下划线连接不同的单词。
使用 static final 修饰一个Java的变量，有一个明显的好处：编译好的 Java 类文件能够加速执行性能。

二、修饰成员方法

效果：被 final 关键字修饰的方法不能被重写。

A final method cannot be overridden

当我们希望所有继承子类都遵循相同的成员方法实现逻辑时，可以使用 final 关键字对成员方法进行约束。

其中，类里的 private 方法会被隐式地指定为 final 方法。
这一点是很自然的，因为 private 方法本来就不能被继承，更不可能被重写。
在成员方法层面，private 等价于 final ，但对于变量而并不是这样。

三、修饰类

使用 final 关键字修饰一个类时，表示这个类不能被继承。

通常会有两种用处：

  1. 针对肯定不能被继承的类，表达其不能被继承，比如 Java 中的 String ，Integer，Float 等类。
  2. 创建一个 不可修改 的类。

其中，为了创建一个 不可修改 的类，通常需要进行以下操作：

  1. 将类声明为 final ；
  2. 将所有成员变量声明为 final ；
  3. 有一个带参数的构造函数；
  4. 所有变量有 getter 方法；
  5. 没有任何 setter 方法

总结

final 关键字可能用于修饰变量、方法和类，主要是为了表达 不可修改 的含义，根据不同的上下文环境可能有功能上的差异。

原理

整型对象 Integer 有 2 种初始化方式：

整型变量有 1 种初始化方式：

这三种初始化的结果：
1）直接创建一个新的值为 100 的 Integer 对象；
2）*获取或创建值为 100 的 Integer 对象；
3）获得一个值为 100 的 int 类型变量；
其中第（2）点由于 Integer 缓存机制的问题，当数值 -128 < x < 127 时会尝试执行类似 firstOrCreate 的操作；当数值在上述范围之外时，一定会创建新的对象。

使用 == 号针对不同的 Integer 对象或 int 变量进行相等判断时，根据等号两侧的情况有不同的逻辑：
1）当 == 号左右两侧都是 Integer 对象时，判断两个对象的内存地址是否相等；
2）当 == 号左右两侧都是 int 变量时，判断两个变量的值是否相等；
3）当 == 号左右两侧分别是 Integer 对象和 int 变量时，判断 Integer 对象的值是否与 int 变量的值相等。

回头看之前的题目：

  • 第1种和第2种情况都是通过初始化方式（2）进行初始化，差别在于目标值是 100 和 128 。根据 Integer 的缓存机制，第1种情况返回的是相同的对象，第2种情况是不同的对象，因此前两种场景的返回值分别是 true 和 false；
  • 第3种情况是通过初始化方式（1）进行初始化，这种情况下无论数值是多少，都会生成新的对象。因此这个场景的比对返回值一定是 false；
  • 第4种情况的比对情况符合比对场景（3），这种情况只需要判断两者的值是否相等，因此返回值为 true；
  • 第5种情况的比对情况符合比对场景（1），由于 i 是通过初始化方式（1）创建，而 j 是通过初始化方式（2）创建，因此两者是不同的对象，返回值为 false 。

总结

根据 三种初始化方式 和 三比对类方式，即可快速判断结果。

初识Rust

Rust是一门系统编程语言，专注于安全，尤其是并发安全，支持函数式和命令式以及泛型等编程范式的多范式语言。Rust在语法上和C++类似，但是设计者想要在保证性能的同时提供更好的内存安全。
Rust最初是由Mozilla研究院的Graydon Hoare设计创造，然后在Dave Herman, Brendan Eich以及很多其他人的贡献下逐步完善的。
Rust的设计者们通过在研发Servo网站浏览器布局引擎过程中积累的经验优化了Rust语言和Rust编译器。
