MS COCO分割标注信息的三种格式转换

背景

在 MS COCO 数据集中，对于分割标注信息的存储可能存在三种格式：

  • Polygon
  • Uncompressed RLE
  • Mask

这三种存储格式之间可以互相转换。在实际使用过程中，会根据计算效率和存储空间的具体情况判定使用哪种格式进行存储及计算。

格式说明

Polygon

多边形存储格式是最直观的一种存储方式，其示例是：

其中包含了一个列表，列表的每个元素是一个长度为 2*k 的数组 P_i，表示一个多边形。这个数组的元素分别是 k 个顶点的坐标，按照数组 P_i 中元素的顺序把这些顶点连接起来，就形成了希望表达的多边形。

在 MS COCO 数据集中，当一个 annotation 对象的 iscrowd 属性为 0 的时候，会使用 polygon 格式存储 segmentation 字段的内容。其实这里隐含着一个信息量，当 iscrowd 为 0 即不那么拥挤的时候，才使用 polygon 格式，反之会使用 RLE 格式，这是因为 polygon 格式的存储量通常会比 RLE 格式的存储量大。

Uncompressed RLE

RLE 是 Run-Length Encoding 的缩写，表示游程编码，是控制论中对于二值图像而言的一种编码方式，能够对连续的二值像素用不同的码字进行编码。其示例是：

对于 MS COCO 数据集而言，RLE 格式默认是 Uncompressed RLE 格式（另外与之相对的有 Compact RLE 格式）。 RLE 格式存储所占字节的大小与边界上的像素数量呈正相关，因此使用 RLE 格式带来的优势在于基于 RLE 格式的存储信息去计算目标区域的面积以及两个目标之间的交并信息时会比较有效率。

这个过程其实是存在模糊定义的。因为一个多边形转为二值矩阵后，要想转换回去，会存在多种对应关系。
举个简单的例子，如果此之多边形的一条边在水平线上且存在多个点，在从二值矩阵往回转换时，是很难精确还原这些顶点信息的。
最朴素的方案是将这个多边形边缘经过的所有像素点都作为一个顶点存储下来，这样的代价是这个 Polygon 格式的存储信息会存在一定冗余。在此基础上可以对同一直线上的点执行合并操作，这是额外可行的优化方案了。
